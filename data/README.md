# Component data directory

This directory holds the **registry and generated artifacts** consumed at runtime.

| Path | Purpose | Committed? |
|------|---------|------------|
| `registry.json` | Central library config (contextLevel, subsumes, typesPath) | Yes (project-specific) |
| `registry.schema.json` | JSON Schema for registry | Yes |
| `metadata/*.json` | Light summaries for catalog MCP | Generated by `pnpm csi:sync` |
| `examples/` | Hand-maintained usage examples | Yes |
| `flattened-types/` | Pre-merged .d.ts for complex components | Generated by `pnpm generate:types` |
| `csi/` | Raw CSI indexer output (build-time only) | Gitignored |

## Quick start

```bash
cp registry.example.json registry.json
# Edit registry.json for your npm packages

pnpm install your-ui-library   # install typed packages
pnpm csi:all                   # index + sync metadata
pnpm generate:types            # flatten high-order component types
pnpm verify:resolver           # smoke test
```

## Using existing data from agent-server

```bash
# Symlink the full component-data tree (recommended for development)
ln -sf ../agent-server/src/agent/tools/component-data ./data-linked
export CSI_DATA_ROOT="$(pwd)/data-linked"
pnpm verify:resolver
```

Or copy once:

```bash
rsync -a ../agent-server/src/agent/tools/component-data/ ./data/
pnpm verify:resolver
```

|----------|---------|-------------|
